API Reference

Complete endpoint documentation for the EcoBot REST API (port 8000) and Admin Panel (port 3001). Use the interactive playground below to send live requests.

Overview

PropertyValue
API Base URLhttp://localhost:8000
Admin Panel URLhttp://localhost:3001
Content Typeapplication/json
AuthenticationX-API-Key header (for /api/* endpoints)
Webhook AuthHMAC secret headers (fail-closed)

Authentication

The EcoBot API uses different authentication methods depending on the endpoint type:

Endpoint TypeAuth MethodHeader
Health checks (/, /health) Public None required
User management (/api/users) API Key X-API-Key: <API_SECRET_KEY>
WhatsApp webhook Webhook Secret X-Waha-Webhook-Secret: <WAHA_WEBHOOK_SECRET>
Telegram webhook Webhook Secret X-Telegram-Bot-Api-Secret-Token: <TELEGRAM_WEBHOOK_SECRET>
Fail-closed webhooks: If the webhook secret environment variable is not set, all requests to that webhook are rejected with 403 Forbidden. You must configure WAHA_WEBHOOK_SECRET and/or TELEGRAM_WEBHOOK_SECRET in your .env.

Endpoint Reference

Health Endpoints

GET /

Root status check. Returns application name and running status.

Auth: Public

Response 200:

{
  "name": "EcoBot",
  "status": "running"
}

GET /health

Lightweight health-check (suitable for load balancers / uptime monitors).

Auth: Public

Response 200:

{
  "status": "healthy"
}

User Management Endpoints

GET /api/users

List all registered users with pagination support.

Auth: API Key — X-API-Key header

ParameterTypeDefaultDescription
limitinteger100Maximum number of users to return
offsetinteger0Number of rows to skip (for pagination)

Response 200:

{
  "users": [
    {
      "phone_number": "628xxx@c.us",
      "username": "John",
      "role": "warga",
      "points": 150,
      "is_active": true,
      "total_messages": 42,
      "total_images": 5,
      "first_seen": "2025-01-15T08:30:00",
      "last_active": "2026-04-28T01:00:00"
    }
  ],
  "total": 1
}

Response 401:

{ "error": "Unauthorized" }

Response 500 (API_SECRET_KEY not set):

{ "error": "Server misconfiguration" }

GET /api/users/{phone}

Get a single user by their phone number identifier.

Auth: API Key

Path ParamDescriptionExample
phonePhone number identifier628123456789@c.us

Response 200: Full user object (same fields as above)

Response 404:

{ "error": "User not found" }

Webhook Endpoints

POST /webhook/whatsapp

WhatsApp (WAHA) webhook receiver. Processes incoming messages and images from WhatsApp users.

Auth: Webhook Secret — X-Waha-Webhook-Secret header

Request Body:

{
  "event": "message",
  "payload": {
    "from": "628123456789@c.us",
    "hasMedia": false,
    "type": "chat",
    "body": "Jadwal pengumpulan sampah hari ini?"
  }
}
StatusBodyMeaning
200{"status": "ok"}Message processed successfully
200{"status": "ignored"}Payload ignored (not a user message)
200{"status": "disabled"}WhatsApp channel is disabled in config
403{"error": "Forbidden"}Invalid or missing webhook secret
500{"status": "error"}Internal server error

POST /webhook/telegram

Telegram Bot API webhook receiver. Processes incoming Telegram updates (messages & images).

Auth: Webhook Secret — X-Telegram-Bot-Api-Secret-Token header

Request Body: Standard Telegram Update object.

{
  "update_id": 123456,
  "message": {
    "message_id": 1,
    "chat": { "id": 12345, "type": "private" },
    "from": { "id": 12345, "first_name": "User", "username": "testuser" },
    "text": "Hello EcoBot!",
    "date": 1714300800
  }
}
StatusBodyMeaning
200{"status": "ok"}Update processed successfully
200{"status": "ignored"}Update type not supported
200{"status": "disabled"}Telegram channel is disabled in config
403{"error": "Forbidden"}Invalid or missing webhook secret
500{"status": "error"}Internal server error

Admin Panel Endpoints (port 3001)

The admin panel runs as a separate Flask app on port 3001. All mutating endpoints are protected by session authentication + CSRF tokens. Rate limiting is applied to the login route.

MethodPathAuthDescription
GET/-Redirect to dashboard or login
GET POST/login-Admin login (rate limited: 5/min, 20/hr)
GET/logoutSessionLogout and clear session
GET/dashboardSessionMain dashboard with statistics
GET/usersSessionUser management list
GET POST/users/createSession + CSRFCreate new user
GET POST/users/<phone>/editSession + CSRFEdit user details
POST/users/<phone>/deleteSession + CSRFDelete user
GET/locationsSessionCollection points list
GET POST/locations/createSession + CSRFCreate collection point
GET POST/locations/<id>/editSession + CSRFEdit collection point
POST/locations/<id>/deleteSession + CSRFDelete collection point
GET/schedulesSessionCollection schedules list
GET POST/schedules/createSession + CSRFCreate schedule
GET POST/schedules/<id>/editSession + CSRFEdit schedule
POST/schedules/<id>/deleteSession + CSRFDelete schedule
GET/analyticsSessionAnalytics and reports page
GET POST/broadcastSession + CSRFSend broadcast to all active users
GET/settingsSessionSystem settings page
GET/api/statsSessionJSON stats for dashboard AJAX
GET/health-Admin panel health check

Testing with cURL

Health Check

curl http://localhost:8000/health

List Users (with API key)

curl -H "X-API-Key: YOUR_API_SECRET_KEY" \
     "http://localhost:8000/api/users?limit=10&offset=0"

Get Single User

curl -H "X-API-Key: YOUR_API_SECRET_KEY" \
     "http://localhost:8000/api/users/628123456789@c.us"

Test Telegram Webhook (rejected without secret)

curl -X POST http://localhost:8000/webhook/telegram \
     -H "Content-Type: application/json" \
     -d '{"update_id":1,"message":{"message_id":1,"chat":{"id":1,"type":"private"},"from":{"id":1,"first_name":"Test"},"text":"hello","date":1700000000}}'
# Expected: 403 Forbidden

Test Telegram Webhook (with valid secret)

curl -X POST http://localhost:8000/webhook/telegram \
     -H "Content-Type: application/json" \
     -H "X-Telegram-Bot-Api-Secret-Token: YOUR_TELEGRAM_WEBHOOK_SECRET" \
     -d '{"update_id":1,"message":{"message_id":1,"chat":{"id":1,"type":"private"},"from":{"id":1,"first_name":"Test"},"text":"hello","date":1700000000}}'

Test WhatsApp Webhook

curl -X POST http://localhost:8000/webhook/whatsapp \
     -H "Content-Type: application/json" \
     -H "X-Waha-Webhook-Secret: YOUR_WAHA_WEBHOOK_SECRET" \
     -d '{"event":"message","payload":{"from":"628xxx@c.us","hasMedia":false,"type":"chat","body":"test"}}'
