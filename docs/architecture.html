System Architecture

How EcoBot is structured: components, data flow, project layout, and technology choices.

High-Level Architecture

Request Flow: User Message → WhatsApp/Telegram → WAHA/Bot API → Webhook Endpoint → Orchestrator → AI Service → Database → Response → User

EcoBot runs as two Flask applications behind Uvicorn (ASGI):

  • API Server (port 8000) — Webhook receivers, user management API, health checks
  • Admin Panel (port 3001) — Web-based dashboard for data management and broadcasting

Both apps share the same PostgreSQL database and configuration.

Data Flow

Incoming Message (WhatsApp)

  1. WAHA receives WhatsApp message

    WAHA forwards the payload as an HTTP POST to /webhook/whatsapp with the X-Waha-Webhook-Secret header.

  2. Webhook verification

    _verify_waha_secret() validates the HMAC header. Rejects with 403 if invalid or not configured (fail-closed).

  3. Channel parsing

    WhatsAppChannel.parse_webhook() extracts sender ID, message type, body text, and media URLs.

  4. Orchestrator routing

    For text: Orchestrator.handle_text() determines intent, fetches conversation context, and calls the AI model.
    For images: Orchestrator.handle_image() downloads media, sends to vision model for classification.

  5. Database interaction

    User lookup/creation, conversation history storage, points update.

  6. Reply delivery

    WhatsAppChannel.send_message() sends the AI response back via the WAHA API.

Incoming Message (Telegram)

Same flow as WhatsApp, but uses /webhook/telegram, X-Telegram-Bot-Api-Secret-Token, and TelegramChannel for parsing and sending.

Security Architecture

  • Webhook HMAC verification — Both channels use hmac.compare_digest() with fail-closed design
  • API key authentication — /api/* endpoints require X-API-Key header with constant-time comparison
  • Admin panel hardening — CSRF tokens (flask-wtf), rate-limited login (flask-limiter), HttpOnly/SameSite cookies
  • SSRF protection — Media downloads validate URL scheme and block private IP ranges
  • Docker — Non-root user, PostgreSQL bound to 127.0.0.1 only
