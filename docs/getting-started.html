Getting Started

Set up EcoBot locally in under 5 minutes. This guide covers prerequisites, installation, configuration, and your first run.

Prerequisites

  • Python 3.10+ with pip
  • PostgreSQL 14+ (local install or Docker)
  • Git for cloning the repository
  • WAHA instance (for WhatsApp) — waha.devlike.pro
  • AI API key — Gemini (free tier available) or OpenAI

Installation

  1. Clone the repository

    git clone https://github.com/mycoderisyad/raflangt-ecobot.git
cd raflangt-ecobot

  2. Create a virtual environment

    # Linux / macOS
python3 -m venv venv
source venv/bin/activate

# Windows
python -m venv venv
venv\Scripts\activate

  3. Install dependencies

    pip install -r requirements.txt

  4. Set up PostgreSQL

    Option A — Docker (recommended):

    docker compose up -d db
# Creates database "ecobot" with user "postgres"

    Option B — Local PostgreSQL:

    createdb ecobot
# Or use pgAdmin to create the database

  5. Configure environment

    cp .env.example .env

    Edit .env and set at minimum:

    # Database
DATABASE_URL=postgresql://postgres:postgres@localhost:5432/ecobot

# AI Provider
AI_PROVIDER=gemini
AI_API_KEY=your-gemini-api-key

# WhatsApp (WAHA)
WHATSAPP_ENABLED=true
WAHA_BASE_URL=http://localhost:3000
WAHA_API_KEY=your-waha-key
WAHA_WEBHOOK_SECRET=a-random-secret-string

# Security
API_SECRET_KEY=change-me-to-a-random-string
ADMIN_PANEL_SECRET_KEY=change-me-to-something-random
ADMIN_PANEL_PASSWORD=a-strong-password

    See Configuration for the full list of environment variables.

Running the Application

API Server (port 8000)

# Development
uvicorn src.app:create_asgi_app --factory --host 127.0.0.1 --port 8000

# Production (multiple workers)
uvicorn src.app:create_asgi_app --factory --host 0.0.0.0 --port 8000 --workers 4

Admin Panel (port 3001)

python admin_panel/app.py
# Access at http://localhost:3001

Docker Compose (full stack)

docker compose up -d
# Starts: PostgreSQL + EcoBot API + Admin Panel

Verify Installation

After starting the server, run these checks:

# Health check
curl http://localhost:8000/health
# Expected: {"status": "healthy"}

# Root endpoint
curl http://localhost:8000/
# Expected: {"name": "EcoBot", "status": "running"}

# Admin panel health
curl http://localhost:3001/health
# Expected: {"status": "healthy", "service": "EcoBot Admin Panel", ...}

Configure Webhooks

WhatsApp (WAHA)

  1. Open your WAHA dashboard
  2. Set webhook URL to http://<your-server>:8000/webhook/whatsapp
  3. Set webhook secret to match your WAHA_WEBHOOK_SECRET env var
  4. Enable the message event

Telegram

  1. Create a bot via @BotFather
  2. Set TELEGRAM_BOT_TOKEN and TELEGRAM_ENABLED=true in .env
  3. Register webhook with Telegram API:
curl -X POST "https://api.telegram.org/bot<BOT_TOKEN>/setWebhook" \
  -H "Content-Type: application/json" \
  -d '{
    "url": "https://<your-public-domain>/webhook/telegram",
    "secret_token": "<TELEGRAM_WEBHOOK_SECRET>"
  }'

First Test

  1. Send help via WhatsApp to your WAHA-connected number
  2. Try AI modes: /hybrid-ecobot, /layanan-ecobot, /general-ecobot
  3. Send a photo of waste to test image classification
  4. Log into admin panel at http://localhost:3001 to see the dashboard
